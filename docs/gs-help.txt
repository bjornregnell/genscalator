genscalator (gs): do-what-I-mean commands. Lead a message with "gs" and I act on your intent,
matched to the nearest command in meaning (not exact syntax).

  gs help                  show this help on the gs do-what-I-mean commands
  gs                       same as gs help
  gs help tt               list all available typed tools and a one-liner of what each does
  gs help tt <what>        show detailed help on the tool nearest in meaning to <what>
  gs tt <tool>             run <tool> and give its output in session, e.g. gs tt chrono
  gs status                the status line information, expanded into a table in session
  gs status line on        turn the status line (line 1) on
  gs status line off       turn the status line off
  gs status mode on        turn the mode line on (line 2: the declared joint state-of-mind)
  gs status mode off       turn the mode line off
  gs status mode           show whether the mode line is on or off
  gs mode                  list the active modes (the recorded joint state-of-mind)
  gs mode add <label>      declare a mode: add a label to the joint state-of-mind
  gs mode rm <label>       clear a mode: remove a label
  gs cues                  list all cues, human -> agent and agent -> human, and what they mean
  gs cue <what>            explain the cue nearest in meaning to <what>
  gs dances                list all dances and what their goals are
  gs dance <what>          explain the dance nearest in meaning to <what>
  gs term <what>           explain the foundations glossary term nearest in meaning to <what>
  gs new app <what> <dir>  seed a complete runnable Scala web app <what> into <dir>
  gs compact notify on     turn the bing-bing on: a notice + chime when a compaction finishes
  gs compact notify off    turn the bing-bing off (silent; the default)
  gs compact notify        show whether the bing-bing is on or off
  gs sound test            play a test chime to check the speakers are on
  gs allow                 set up the recommended tt allowlist for this repo (shown; you approve)
  gs help allow            explain the recommended allowlist and its principles (no changes)

  gs reqt [<file>]         parse + lint a reqT-lang file (default PRD.md); report both
