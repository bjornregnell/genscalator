# Terminal Sudoku — Product Requirements (reqT-lang)

## Goal level

* Goal: playAndSolve has
  * Spec: A user can play a Sudoku puzzle in the terminal and have it solved on demand.
  * Why: the benefit is practice plus reliable assistance.
* Goal: trustworthySolving has
  * Spec: The solver always finds a solution when one exists, so the user can trust its help.

## Domain level

* Feature: terminalPlay has
  * Spec: A user loads a puzzle, views the grid, enters digits into empty cells, and requests a solve, all via terminal input and output.
* Feature: rejectIllegalMoves has
  * Spec: The product rejects a move that violates Sudoku row, column or box rules, so the user cannot enter an illegal board.
* Feature: terminalPlay helps Goal: playAndSolve
* Feature: rejectIllegalMoves helps Goal: trustworthySolving

## Product level

* Function: backtrackingSolver has
  * Spec: The product solves a nine by nine board by backtracking search, returning the solved grid or a not-solvable result.
* Function: boardIo has
  * Spec: The product loads a board from an eighty-one character text form (0 or dot for an empty cell) and renders a board back to text.
* Function: legalPlacement has
  * Spec: The product reports whether placing a digit in a cell is legal under the row, column and box rules.
* Function: backtrackingSolver helps Feature: terminalPlay
* Function: legalPlacement helps Feature: rejectIllegalMoves
* Function: boardIo helps Feature: terminalPlay

## Design level

* Design: scalaCliTerminal has
  * Spec: Scala 3.8, runnable via scala-cli; terminal input and output only, no framework.
* Design: immutableBoard has
  * Spec: The board is an immutable value; solving returns new boards, never mutating a shared grid in place.
* Design: cellAdt has
  * Spec: A cell is modeled with a Scala 3 enum or ADT distinguishing Empty from a Filled digit, handled by pattern matching; no magic zero-means-empty sentinel scattered through the logic.
* Design: testApproach has
  * Spec: mUnit tests: the solver solves known puzzles and the result is a valid complete grid matching the expected solution for a unique-solution puzzle; legalPlacement accepts and rejects known cases; boardIo round-trips a board. The interactive loop is not unit-tested.
* Design: scalaCliTerminal helps Function: backtrackingSolver
* Design: immutableBoard helps Function: backtrackingSolver
* Design: cellAdt helps Function: backtrackingSolver
* Design: testApproach verifies Function: backtrackingSolver
* Design: testApproach verifies Function: legalPlacement
* Design: testApproach verifies Function: boardIo
