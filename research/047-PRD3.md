# CLI word-frequency tool — Product Requirements (reqT-lang)

## Context

* Product: wordFreq has
  * Gist: A command-line tool that reports the most frequent words in a text file.
* Stakeholder: analyst has
  * Gist: Someone inspecting a text file who wants its top word frequencies.

## Product level

* Function: topNWords has
  * Spec: Given a text file path and an integer N, the product prints the N most frequent words with their counts, most frequent first; count ties are broken alphabetically (ascending).
* Function: tokenize has
  * Spec: The product splits text into words case-insensitively and ignores surrounding punctuation, so The and the-with-a-comma count as the same word.
* Function: cliArgs has
  * Spec: The product reads the file path and N from command-line arguments and prints a one-line usage message on a missing or invalid argument.

## Design level

* Design: scalaCliFile has
  * Spec: Scala 3.8 via scala-cli; reads a file and writes to standard output; no dependency beyond the Scala toolkit.
* Design: collectionPipeline has
  * Spec: Frequency counting uses an immutable collection pipeline (groupBy or a fold to a Map, then sort), not a mutable counter loop; the result is an immutable sequence of word and count pairs.
* Design: optionArgs has
  * Spec: Argument parsing returns Option or Either and prints a usage message on failure; no exceptions for control flow and no null.
* Design: testApproach has
  * Spec: mUnit tests over the pure core (tokenize and top-N as functions on a String, separated from file reading and printing): a known text yields a known top-N; ties are broken deterministically; empty input yields an empty result.
* Design: collectionPipeline helps Function: topNWords
* Design: scalaCliFile helps Function: topNWords
* Design: optionArgs helps Function: cliArgs
* Design: testApproach verifies Function: topNWords
* Design: testApproach verifies Function: tokenize
