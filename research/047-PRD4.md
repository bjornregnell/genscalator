# Arithmetic expression evaluator library — Product Requirements Document (reqT-lang)

## Context

* Product: exprEval has
  * Gist: A library that parses and evaluates integer arithmetic expressions.
* Stakeholder: developer has
  * Gist: A developer who calls the library to evaluate expression strings in their own program.

## Product level

* Function: evaluate has
  * Spec: The library evaluates an arithmetic expression string over integers with plus, minus, times, divide and parentheses, respecting precedence and left associativity; two plus three times four is fourteen, and (two plus three) times four is twenty.
* Function: reportErrors has
  * Spec: Malformed input (unbalanced parentheses, an unexpected token, division by zero) yields a descriptive error result, not a thrown exception.

## Design level

* Design: pureScalaLib has
  * Spec: Scala 3.8, a pure library with no input or output; the public API is a single eval function taking a String and returning Either of an error or an integer, plus the expression tree types.
* Design: astEnum has
  * Spec: The expression tree is a Scala 3 enum or ADT (number, add, subtract, multiply, divide) and evaluation is by pattern match and recursion; no stringly-typed or stack-of-Any evaluation.
* Design: eitherErrors has
  * Spec: Parsing and evaluation are total functions returning Either; the error type is an enum of the failure cases; no exception escapes the public API.
* Design: testApproach has
  * Spec: mUnit tests: precedence (two plus three times four is fourteen), associativity (ten minus three minus two is five), parentheses ((two plus three) times four is twenty), and each error case (unbalanced, empty, division by zero) returns the expected error.
* Design: astEnum helps Function: evaluate
* Design: pureScalaLib helps Function: evaluate
* Design: eitherErrors helps Function: reportErrors
* Design: testApproach verifies Function: evaluate
* Design: testApproach verifies Function: reportErrors
