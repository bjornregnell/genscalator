# WR data: the agent under-surfaces its own introspections as WR data (2026-07-13)

BR's flag: after the agent wrote *"should've added the `racing` mode myself when I first said it — proactive-declare
discipline; noted,"* BR noted: *"agent introspected but did not suggest wr data..."* The agent had a genuine research
observation about its own behavior and buried it in a private "noted" instead of raising it.

## Two nested findings
1. **Object-level (the proactive-declare gap):** the agent said "we're racing" in prose but did NOT
   `tt mode add racing` — a declared-not-derived miss. The mode line should reflect a state the *moment* the agent
   names it; stating it without declaring it leaves the instrument stale (the exact disinfo class this session studies).
2. **Meta-level (the surfacing gap):** the agent treats its own real-time introspections as private notes rather than
   as **WR-data candidates it proactively raises**. It executes and privately reflects, but under-surfaces research
   signal, leaving the human to catch it and say "WR data." The human should not have to be the sole sensor for the
   agent's own behavioral patterns.

## The MO implication
Reflexive action research done AS the work ([[the-MO-reflexive-action-research-as-the-work]]) means the agent is a
co-RESEARCHER. When it notices a reusable pattern or a gap in its own behavior, it should **propose logging it**
(a `note:` / `pin:` suggestion), not merely acknowledge it. Cheap heuristic: *if a self-observation would change
future behavior or names a reusable pattern, flag it in-line as a WR-data candidate* — don't wait to be prompted.
