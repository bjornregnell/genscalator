# CO4 forgot the box was flaky - memory loss across compactions (2026-07-09)

**Observation (BR, WR data 2026-07-09):** CO4 had forgotten that the box (blixten) is flaky and that it crashed a few days ago. BR had to re-flag it mid-session ("remember flaky box"). Not a big deal operationally, but - in BR's words - "yet another evidence of memory loss (probably due to many compactions)."

**Why it is data:** this engagement is RT047, the longitudinal live case study. The flaky-box fact was established earlier in the study, then dropped out of CO4's working memory - the loss happened during this session (pre-compact context was 93% full: 903k / 1M message tokens). **BR's refinement of the mechanism (2026-07-09):** the cause may be **compaction** (the fact literally dropped from the summarized context) OR **big context fill / rot** (the fact still present in-window but not attended-to at high fill - attention dilution rather than lossy compression). These are distinct failure modes worth separating; the 93% fill makes the fill/rot mechanism plausible even before the compact fired. It is a clean specimen of the memory-loss / context-rot failure mode the study tracks: durable facts silently falling out of the agent's head, surfaced only when the human notices the agent acting as if it never knew. The asymmetry is the point - the human caught it; the agent did not know it had forgotten.

**Fix applied:** pinned a durable memory `blixten-box-flaky` so the fact survives future warps (the durable-substrate-as-memory-prosthesis pattern - the same reason the PIN-BOARD and resume prompt exist). Recurrence test: a future compacted CO4 should recall the flaky box from the memory index without BR having to re-flag it.

**Ties:** RT047 (longitudinal case study), the context-rot thread, [[joint-rot-vigilance-recovery-kit]], and the resume-prompt / PB substrate-as-prosthesis design. Companion to the observation that memory (the ~/.claude store) is the ONE channel that survives compaction, so study-critical facts belong there, not only in the (compactable) conversation.
