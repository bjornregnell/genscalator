# Externalizing an intention aids memory at request-time — and latent-vs-hot knowledge (2026-07-16)

**Type:** WR data — agent modelling human cognition + a note on latent-vs-hot knowledge. BR: *"may help you understand
how humans work (but you maybe already 'knew' such stuff even if not hot in context)."*
**Threads:** SM124 (`agentHelpHumanNotForget` + `humanWellbeing`), the reminder feature, the human-state line,
the context-rot / substrate spine.

## The reasoning

BR asked the agent to remind him at 13:00, then noted the reminder could stay session-only because **the act of
asking already made him not-forget** — *"that is how humans work."* The agent generalised: **externalising an
intention (handing it to a reminder, a note, or the agent) aids memory at REQUEST time, not only at fire time** — the
value lands the moment the intention is offloaded, before the reminder ever fires. So `agentHelpHumanNotForget`
(SM124) helps in two moments: the **offload** (now) and the **reminder** (later).

## Echt caveat: SURFACED, not invented

This is **established cognitive science**, not a novel agent discovery — the *extended mind* (Clark & Chalmers),
*cognitive offloading*, *implementation intentions* (Gollwitzer), the *Zeigarnik effect* (open loops nag until
externalised or resolved). BR's meta-point: the agent **very likely already "knew" this latently** (it is in the
training corpus) even though it was **not hot in context** — it emerged through the reasoning rather than being
recalled as a fact. So this is an agent **re-surfacing** known human-cognition, not contributing new science. Do not
over-claim it (echt).

## Why it matters (two things)

1. **Better human-modelling → better help.** If offloading is *itself* the aid, the reminder's *timing* is secondary
   and the *prompt acceptance* of the intention is already valuable — which shapes how `agentHelpHumanNotForget` and
   the wellbeing nudges should behave (accept + acknowledge the intention promptly, not merely schedule it).
2. **The latent-vs-hot distinction is the project's spine, seen from the agent side.** "Knew it but not hot in
   context" is exactly the context-rot / substrate theme: **training-latent** knowledge vs **context-active**
   knowledge behave differently, and the whole genscalator method (substrate, `gs warm`, resume-prompts) is about
   keeping the *right* knowledge hot. Datapoint: much useful agent "insight" is **re-surfacing latent knowledge into
   context**, not generating novelty — worth remembering when judging what counts as an agent contribution.
