# A human deliberately disclosing their state-of-mind ("I'm stressed") is high-value data (2026-07-11)

BR-flagged, meta. When a human **deliberately** tells the agent their affective/cognitive state — *"BR is stressed"*,
*"pin it for later when BR less stressed"*, *"a couple of minutes before afk"* — that is **important information for
the agent's understanding of the human's state of mind**, to be treated as high-signal, not incidental.

## Why it matters
The agent **cannot perceive the human's affective state directly.** This is the *reverse* of the corroboration
asymmetry: on the human's OWN state the human is the **ground-truth reporter** (self-report here is authoritative, not
confabulation-prone). So a deliberate disclosure is the agent's **only reliable channel** to that state — the human
instrumenting the joint system with data the agent otherwise lacks. Under-weighting it wastes the highest-quality
state signal available.

## What the agent should DO on a stress disclosure
- **Reduce the human's load:** fewer/no decisions at once, defer non-urgent asks, batch, simplify prose, cut optional
  detail (educate-lead-with-one-screen, harder).
- **Protect control + triggers:** don't push, don't pile on pastes/modals, favor safe-stop / hold.
- **Treat it as a mode-selection input** — toward saving/rest, away from spend-heavy / high-throughput (ties TSM/HHM
  in blog 020; the rest dance; the joint-zone in `research/011`).
- **Persist it as a real gate** — BR did exactly this: defer the settings paste until less stressed.

## "human-stress-mode" — a named human mode (BR, 2026-07-11)
BR: *"for our status awareness: a new human mode: human-stress-mode."* So the human's stress state gets a NAMED mode
in our shared mode vocabulary (alongside TSM/HHM in blog 020), so either party can name it in one cue and the agent
adapts per the rules above.

**Agent WDYT (pinned here per BR, not in chat):**
- **Yes — naming is the cheap coordination primitive** (the blog-020 thesis). "human-stress-mode" lets BR flag his
  state in one token and the agent switch behaviour deterministically (reduce load, defer, simplify, safe-stop)
  without spelling out the adaptation each time.
- **It is the HUMAN-side counterpart** to the token/agent modes (TSM = spending, HHM = harvest). The vocabulary is
  becoming two-sided: **agent/token modes** (what we do with tokens) + **human-state modes** (the human's
  affective/cognitive state). Both feed mode-selection — a clean symmetry for the super-context.
- **Who names it / entry:** either party. BR self-reports (ground truth); the agent may *observe* signals (rushed
  messages, rising typos, "racing", "a couple of minutes") and **propose** "are we in human-stress-mode?" — but the
  human's member-check confirms (agent proposes, human ratifies). **Exit cue (BR, 2026-07-11): *"not so stressed now"***
  ends the stress period (exits human-stress-mode). So the mode has a clean entry (a stress disclosure) + a clean exit
  ("not so stressed now") — a bounded period the agent can track.
- **What it triggers = the stress-note rules above.** human-stress-mode is the named *trigger*; the behaviours
  (reduce load, defer, safe-stop, saving/rest lean, protect control) are what it *fires*.
- **Distinct from `:Z` (tired), same family.** Different adaptations: tired → rot-vigilance + steer to safe stop;
  stressed → reduce-load + defer + protect triggers. Keep them as distinct cues under one "human-not-at-optimum"
  family (tired · stressed · time-pressured).
