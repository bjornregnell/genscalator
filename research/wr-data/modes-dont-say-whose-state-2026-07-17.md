# The mode vocabulary doesn't say WHOSE state — and `human-stress` is the only one that does

**2026-07-17 00:38 (`tt chrono now`).** BR's finding, ratified by him for WR data (*"good finding; pin that to the
related tasks"*). Fourth instance of one defect in a single night. **SM134 #6 · SM085 (terminology) · SM121.**

## The trigger — BR, on the cue vocabulary

> *"and the suggested preprompt also conflates this `go afk safe solo` — **YOU CANNOT go afk you dont have a
> keyboard**: so I'd better say: `go safe solo +afk` (or ***me*** go afk not you)"*

**He is right, and it is grammatical, not stylistic.** `go afk safe solo` makes **the agent the subject of a human
predicate.** AFK means *away from keyboard*. **The agent has no keyboard.** It cannot be away from one. The cue
attaches a fact about **where BR is** to a verb aimed at **the agent**.

His fix separates what was fused: **`go safe solo`** = an **authorization** (about the agent) · **`+afk`** = a
**state declaration** (about the human).

## The finding

`tt mode --help` defines a mode as *"a label on the shared human↔agent state."* **Shared.** But:

> ## ⇒ **The vocabulary never says WHOSE state a label describes.**

| mode | whose state? | names its subject? | can BOTH parties hold it? |
|---|---|---|---|
| `afk` | **the human's** | ❌ | no — only the human has a keyboard |
| `human-stress` | **the human's** | ✅ **yes** | no |
| `hangover?` | *the agent's* (intended) | ❌ | ⚠️ **YES — a human can be hungover, literally** |
| `high-context` | the agent's | ❌ | no — humans have no context window |
| `rot-vigil` | joint | n/a | n/a |
| `tok-spend` / `token-saving` | joint (the budget) | n/a | n/a |
| `solo` | joint (the arrangement) | n/a | n/a |
| `racing` | joint (the interaction) | n/a | n/a |

⭐ **`human-stress` is the ONLY non-joint mode that names its subject — and it is the ONLY one that has never gone
wrong.** That is not a coincidence; it is the control group, and it was sitting in the vocabulary the whole time.

## TWO independent missing attributions (do not merge them — SM134 #3 is the other one)

1. **PROVENANCE — *who SET this?*** → SM134 #3. A chip that decayed looks identical to one somebody cleared.
2. **SUBJECT — *WHOSE STATE does this describe?*** → this note.

**They are orthogonal, and the gap between them is exactly where tonight's errors lived:** the agent **set** `afk`
(provenance: agent) about **BR's** state (subject: human) — **a mode it is structurally incapable of knowing.**

## Three distinct failure paths, all observed tonight

1. **Ambiguous subject — `hangover?`.** *Both* parties can plausibly be hungover, so the label is genuinely
   ambiguous. And the chip **fired on BR's own thinking pause**: it rendered the human's silence as the agent's
   state (`hangover-chip-fires-on-the-humans-thinking-pause-2026-07-17.md`). **Whose hangover was it reporting? The
   label cannot say.**
2. **Wrong grammatical subject — `go afk`.** The cue's verb takes the agent; the predicate belongs to the human.
3. **Wrong setter — the agent adding `afk`.** ⚠️ **Twice in twenty minutes** (00:3x and 00:4x, both reverted). Both
   times the agent **inferred BR's physical location from his message content.**

## ⇒ The structural rule: only the HUMAN may declare `+afk`

**Not a convention — a CAPABILITY boundary.** [[agent-blind-to-input-channel-and-timing]] already says it:
*"the agent sees only the CONTENT of input, not its channel/keystrokes/timing; **never infer how/when/where the human
typed**."*

**The agent broke that memory twice tonight.** It was **carried and not armed** — the exact claim this session is
testing. ⭐ **BR's fix makes the memory unnecessary: a mode only the human can set cannot be wrongly set by the
agent.** In-agent rule ⇒ rots. Capability boundary ⇒ holds. **The day's load-bearing finding, applied to the
vocabulary itself.**

**✅ First outing, 00:38 — it worked.** BR typed `+afk` himself. The agent ran `tt mode add afk` **as a transcription
of a declared fact, not as an inference.** No guessing was involved or possible.

## The pattern this completes — FOUR instances, one night, one defect

| # | instance | who mis-attributed what |
|---|---|---|
| 1 | mode line has no provenance (23:47) | BR credited the agent with a drop that never happened |
| 2 | `hangover? 1m` (00:13) | the chip rendered **BR's pause** as **the agent's** state |
| 3 | agent added `afk` ×2 (00:3x, 00:4x) | the agent asserted **BR's location** from message content |
| 4 | the `go afk` cue | the vocabulary makes **the agent** the subject of a **human** predicate |

**All four are the same shape: the shared-state surface does not track WHOSE state it is, or WHO said so.** It was
built to be *shared*, and sharing turned out to mean *unattributed*.

## Candidate (SM134 #6) — NOT built

**Clears the anti-quota gate:** four dated specimens in one night.

- **Sketch:** name the subject when a label is **not joint** and **both parties could plausibly hold it** —
  following `human-stress`, which already proves the pattern. `afk` → `human-afk`? `hangover?` → `agent-hangover?`
  ⚠️ **but weigh it against terseness: the mode line is a scarce, glanceable surface** and `human-` prefixes are
  expensive. **`high-context` needs no prefix** (humans have no context window), so the rule is **not** "prefix
  everything" — it is *disambiguate only where both could hold it*.
- **Cheaper alternative, possibly the whole fix:** leave the labels and fix **the cue** (BR's `go safe solo +afk`)
  plus **the setter rule** (only the human declares his own state). ⭐ **Consistent with tonight's other lesson: the
  agent proposed a redesign, BR proposed a rename** (SM134 #5).
- ⛔ **NOT built. `afk` is declared; renaming live modes mid-session would change the instrument** while it is
  measuring. **BR ratifies vocabulary** (`research/035-go-verb-vocabulary.md`).
