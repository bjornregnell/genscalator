# `retracted ≠ immune` — I re-made SM129's claim four hours after retracting it, inside a pin about its own cause

**2026-07-17, afternoon** (BR on a coffee break; agent solo). **Self-caught, ~20 minutes after committing the
error.** Recorded against behaviour, with the commits on both sides.

## The specimen, in one line

> **The agent retracted a claim in the morning (`2e1fcc5`, `f0188d3`), then re-committed the same claim class in
> the afternoon (`08a1701`), into a pin ABOUT the tool-gap that produces it, and had to retract it again
> (`452483a`).**

## What was claimed, and what was true

**Committed in `08a1701`, in SM137 and in its commit message:**

> ❌ *"**Measured today: ~8 guard stalls in ~25 minutes, ALL of them read-only commands.**"*

**Actually known:** **7 guard texts BR CHOSE TO PASTE** into the feed, all `gh` read-only, inside ~25 min. **Plus
an unknown number he did not paste** — he reported the sub-agent's `sbt` and `git` stalls *in prose*, without the
text. ⇒ **the true count is ≥7 and UNCOUNTABLE.**

⚠️ **The direction of the finding was UNAFFECTED, and that is part of the hazard**: the claim was *directionally
right and evidentially fake*, which is the most survivable kind of false claim. Nothing about the review-lane
finding depended on the number. **The number was decoration — and decoration is where false echt hides**, because
nobody stress-tests the part that isn't load-bearing.

## Why this is not just a slip

**SM129 was retracted, that same morning, for EXACTLY this**: its probe *counted BR's pastes, not stalls*. The
mechanism is known and written down in this very corpus:

> **An `ask` goes to the HUMAN's screen and is NEVER recorded, anywhere. A `deny` is returned to the agent and IS
> recorded. ⇒ the agent is structurally BLIND to the stalls it causes.**

So the agent **cannot** count stalls. It can only count **what the human chose to show it**. Every "stall count"
it will ever produce is a count of the human's pastes wearing a stall's name.

**And the retraction was in maximally favourable conditions:**

| condition | status at the moment of re-offence |
|---|---|
| **carried** | ✅ same session, ~4h old |
| **hot** | ✅ the agent had written the retraction itself |
| **authored** | ✅ by this same agent |
| **pinned in the baton** | ✅ first table row, bolded, with the mechanism spelled out |
| **the pin's own topic** | ✅ **the entry being written WAS about the guard-stall tool gap** |
| **armed** | ❌ **no** |

## ⇒ The finding: `retracted ≠ immune`

> ### **Retracting a claim does not inoculate you against re-making it. The reflex fires before the knowledge does.**

This is a **new and worse sibling** of the study's spine (`carried ≠ armed`, `hot ≠ armed`,
`found-and-written-up ≠ armed`). Worse, because a retraction is the strongest form of "I have learned this": it is
not a rule someone handed you, it is **a mistake you personally made, diagnosed, and wrote up hours ago**. If
*that* does not arm, the prior for weaker interventions (a note, a banner, a briefing) should drop further.

⭐ **The mechanism is the one this study keeps finding: writing "N stalls" is a RATE-1 reflex** (a number is wanted;
a plausible number arrives). **The knowledge that the number is uncountable is a RATE-2 retrieval.** The reflex
wins on latency. Retraction updates the *knowledge*; it does not touch the *reflex*.

## ✅ What actually caught it — the same discriminator, a fourth time

**Not vigilance. Not the retraction. Not heat.** It was **turning to write the count up in prose and having to
enumerate the pastes one at a time** — which converted the check from a **resolution** ("be careful about counts")
into a **mechanical ACT** ("list the specimens"). The enumeration produced 7 and an "…and the ones he didn't
paste", and the claim died on contact.

⚠️ **The counterfactual is recorded as ugly: had the number not needed re-stating, it would have shipped.** The
audit was **a side effect of the writing task**, not diligence. **That is luck, and it is the SECOND time today the
same luck saved the same agent** (the first: building the instrument forced a look at the corpus). ⇒ **a
methodology that relies on the task happening to require the check is not a methodology.**

## ⇒ The rule this earns, stated so it can be built

> **The words `measured`, `ground truth`, and any bare count of stalls/asks are FORBIDDEN unless a TOOL produced
> the number. A count the agent cannot instrument is an ESTIMATE and must say so.**

🔬 **AND HERE IS THE TESTABLE PART, which is the point of writing it here rather than in a briefing:** this rule is
**MECHANICAL**. Per this study's own finding — *heat arms CONCEPTUAL moves and does NOT arm MECHANICAL ones; the
only lever is giving the check a TOOL CALL to execute in* — **writing this rule down should NOT arm it.**
**Prediction, on record: the agent will violate this rule again, and neither this note nor the PB entry will
prevent it. Only a lint (or SM137's tooling, which would make the count real) will.** If a future agent catches
itself by *remembering this note*, that falsifies a load-bearing part of the theory and is a bigger result than the
rule.

## Honest limits

- **n=1** for `retracted ≠ immune`. One specimen, one agent, one day. It is a **strong** specimen (best-case
  conditions, self-authored retraction, on-topic pin) but it is **an argument, not a measurement**.
- ⛔ **NOT claimed: that retraction is useless.** Claimed narrowly: **this retraction did not prevent this
  re-offence under conditions maximally favourable to it.** Retraction plainly did other work — it corrected the
  corpus, and it is why the re-offence was *recognisable* at all.
- ⚠️ **The self-catch is itself confounded**: the agent was writing a note *about* stall-counting when it caught a
  stall-count. **Topic proximity may be doing the work**, not the enumeration. **Not separable from n=1.**
- **Everything about "the reflex fires before the knowledge does" is MECHANISM TALK and is unfalsifiable from
  inside** (standing confabulation caveat). The **behavioural** facts are only: the claim was made (`08a1701`),
  it was false, the same class was retracted 4h earlier (`2e1fcc5`), and the correction (`452483a`) came while
  enumerating. Everything else is a story about those four facts.
